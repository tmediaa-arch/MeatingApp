# راهنمای راه‌اندازی و اجرای سرویس پیامک (کاوه‌نگار)

این سند نحوهٔ پیکربندی سرویس پیامک، اجرای صف (queue) و زمان‌بند (scheduler)
را روی سرور اختصاصی (Supervisor) و هاست اشتراکی (Cron) توضیح می‌دهد.

---

## ۱. پیکربندی اولیه

### ۱.۱ متغیرهای محیطی (`.env`)

```env
# کاوه‌نگار
KAVENEGAR_API_KEY=کلید-API-پنل-کاوه‌نگار
KAVENEGAR_OTP_TEMPLATE=otp-login
KAVENEGAR_INVITE_TEMPLATE=invite-link

# صف — sync (بدون worker) یا database (نیازمند worker)
QUEUE_CONNECTION=sync
```

### ۱.۲ تعریف الگوها در پنل کاوه‌نگار

متد اعتبارسنجی (`verify/lookup`) فقط با **الگوی تأییدشده** کار می‌کند. دو الگو بسازید:

| الگو | نام پیشنهادی | نمونهٔ متن |
|---|---|---|
| کد ورود | `otp-login` | `کد ورود شما: %token` |
| لینک دعوت | `invite-link` | `دعوت به سامانهٔ جلسات: example.com/invite/%token` |

> در الگوی دعوت، **آدرس دامنهٔ سامانه را داخل متن الگو بنویسید**؛ سامانه فقط
> مقدار `%token` (شناسهٔ دعوت) را ارسال می‌کند، چون متد verify اجازهٔ ارسال
> توکن دارای `/` یا فاصله را نمی‌دهد.

---

## ۲. حالت ساده — بدون worker (پیش‌فرض)

با `QUEUE_CONNECTION=sync` پیامک‌ها **بلافاصله در همان درخواست** ارسال می‌شوند.
هیچ worker یا Cron اضافه‌ای برای پیامک لازم نیست. این حالت برای شروع و
هاست‌های ساده مناسب است.

> با این حال، برای **بقیهٔ کارهای زمان‌بندی‌شدهٔ سامانه** (Escalation وظایف،
> retry اعلان‌ها، تیک‌های گردش کار و ...) همچنان باید زمان‌بند را فعال کنید —
> بخش ۵ را ببینید.

---

## ۳. حالت پیشرفته — صف database (ارسال غیرهمزمان)

برای اینکه پیامک‌ها در پس‌زمینه ارسال شوند و سرعت پاسخ‌گویی بالا بماند:

```bash
# ۱) ساخت جدول‌های صف (یک بار)
php artisan migrate

# ۲) تغییر درایور صف در .env
QUEUE_CONNECTION=database
```

سپس باید یک **worker** اجرا شود (بخش ۴).

---

## ۴. اجرای worker صف

### ۴.۱ سرور اختصاصی / VPS — با Supervisor (توصیه‌شده)

فایل `/etc/supervisor/conf.d/meetingapp-worker.conf`:

```ini
[program:meetingapp-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/meetingapp/artisan queue:work --sleep=3 --tries=3 --max-time=3600
autostart=true
autorestart=true
stopaswaitsecs=3600
user=www-data
numprocs=2
redirect_stderr=true
stdout_logfile=/var/www/meetingapp/storage/logs/worker.log
```

فعال‌سازی:

```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start meetingapp-worker:*
```

> پس از هر بار Deploy، `php artisan queue:restart` را اجرا کنید تا workerها
> کد جدید را بارگذاری کنند.

### ۴.۲ هاست اشتراکی — با Cron

روی هاست اشتراکی معمولاً Supervisor در دسترس نیست. یک Cron تعریف کنید که
هر دقیقه صف را پردازش کند و سپس خارج شود:

```cron
* * * * * cd /home/USER/meetingapp && php artisan queue:work --stop-when-empty --max-time=55 >> /dev/null 2>&1
```

- `--stop-when-empty` : پس از خالی‌شدن صف خارج می‌شود.
- `--max-time=55` : حداکثر ۵۵ ثانیه کار می‌کند تا با اجرای دقیقهٔ بعد هم‌پوشانی نکند.

---

## ۵. اجرای زمان‌بند (Scheduler) — الزامی

سامانه چند فرمان زمان‌بندی‌شده دارد (Escalation وظایف، retry اعلان‌ها،
تیک گردش کار، اجرای گزارش‌های زمان‌بندی‌شده و ...). برای اجرای آن‌ها **یک**
Cron کافی است — چه روی VPS و چه روی هاست اشتراکی:

```cron
* * * * * cd /path/to/meetingapp && php artisan schedule:run >> /dev/null 2>&1
```

این تنها Cron موردنیاز سامانه است؛ بقیهٔ زمان‌بندی‌ها داخل `routes/console.php`
مدیریت می‌شوند.

---

## ۶. مدیریت Jobهای ناموفق

```bash
php artisan queue:failed        # فهرست Jobهای ناموفق
php artisan queue:retry all     # تلاش مجدد همه
php artisan queue:flush         # پاک‌کردن همه
```

`SendSmsJob` به‌صورت خودکار تا ۳ بار با فاصلهٔ ۱۵، ۶۰ و ۱۸۰ ثانیه تلاش مجدد می‌کند.

## ۷. آزمایش سریع

```bash
php artisan tinker
```

```php
// ارسال یک کد آزمایشی
app(\App\Domains\Identity\Services\OtpService::class)->send('09123456789');

// ارسال مستقیم پیامک
app(\App\Domains\Notifications\Services\KavenegarSmsService::class)
    ->sendVerify('09123456789', '12345', config('services.kavenegar.otp_template'));
```

اگر `QUEUE_CONNECTION=database` است، برای پردازش فوری در محیط توسعه:

```bash
php artisan queue:work --once
```
